Organize AutoPilot devices in dynamic AAD groups using GroupTags & PowerShell

Published: 12 Dec 2019
File under: Intune PowerShell

Don‚Äôt ever say Microsoft doesn‚Äôt listen! One of my biggest pet peeves was solved at the beginning of the month when Microsoft announced the ability to edit device group tags!

This doesn‚Äôt sound like much, but it essentially unlocks the potential of group tags that was never really there before - we can now use group tags to dynamically control device group membership.

While there have been quite a few great posts about this, most of the code in the articles I‚Äôve read are using community modules that, while there‚Äôs nothing wrong with using other people‚Äôs code, I find the best way to understand how something works is to dig in and figure out the inner-workings myself.

The good news is that because almost *everything* in Azure is accessible through the omnipresent Microsoft Graph, so it only took a few minutes of analyzing the requests to figure out what was going on. Let‚Äôs dive right in!

First let‚Äôs set up a few dynamic groups in AAD.

The first one - let‚Äôs call this our ‚Äúbaseline‚Äù group, will have the following dynamic rule that will capture all devices.

(device.devicePhysicalIDs -any _ -contains "[ZTDId]")

The second group - let‚Äôs call this our ‚Äúexclusion‚Äù group, will have the following dynamic rule that will capture all devices with a specific group tag.

(device.devicePhysicalIds -any _ -eq "[OrderID]:EXCLUDE")

Once set up and given enough time to update, you should now see all of your devices in the ‚Äúbaseline‚Äù group and nothing in the ‚Äúexclusion‚Äù group. Let‚Äôs fix that now.

Go and grab the serial numbers of the devices you wish to move to the exclusion group (you can grab this directly from AAD or from the Windows AutoPilot Devices section of Intune - here‚Äôs mine as an example - note that there are no group tags assigned)

For this example, let‚Äôs exclude the first device in the screenshot above.

The first step when we interact with Microsoft Graph, as always, is to authenticate and store the Authentication Token in a variable so that we can use it for the next few steps - I‚Äôve spoken about this before, so if you aren‚Äôt sure how to do this, read this and come back when you are ready!

$authToken = Get-AuthToken #use one of a million ways to get the authtoken...

Alright - we‚Äôve got our auth token, put your serial number into a variable, and then we will form out the first Graph Call.

$serialNumber = "Put Your Serial Number Here"
$graphUri = "https://graph.microsoft.com/beta/deviceManagement/windowsAutopilotDeviceIdentities?`$filter=contains(serialNumber,'$serialNumber')"
$result = (Invoke-RestMethod -Method Get -Uri $graphUri -headers $authToken).value

If we‚Äôve formed our request properly, if we look at the contents of $result, we should see details on the device in question.

Next, we are going to grab the id of the device from the graph call, build the group tag data and post it back to Graph..

$id = $result.id
$body = @{
    groupTag = "EXCLUDE"
}
Invoke-RestMethod -Method POST -Uri "https://graph.microsoft.com/beta/deviceManagement/windowsAutopilotDeviceIdentities/$id/updateDeviceProperties" -Body ($body | ConvertTo-Json -Compress) -Headers $script:authToken

Finally, we are going to perform a sync on our AutoPilot device list..

Invoke-RestMethod -Method Post -Uri "https://graph.microsoft.com/beta/deviceManagement/windowsAutopilotSettings/sync" -Headers $script:authToken

Now if we jump over to our AutoPilot devices list we should see the group tag updated..

And of course, our dynamic groups should now be populated correctly..

That‚Äôs it! - 3 calls to Graph and a little bit of patience.

Now of course, 3 REST API calls is not how I‚Äôm going to leave you - with a little bit of error handling and polish we can build out a very reliable function..

function Update-AutoPilotGroupTag {
    [cmdletbinding()]
    param (
        [parameter(Mandatory = $true)]
        [string[]]$deviceSerial,

        [parameter(Mandatory = $false)]
        [string]$groupTag,

        [parameter(Mandatory = $false)]
        [switch]$sync
    )
    try {
        if (!($script:authToken)) {
            $script:authToken = Get-AuthToken -user $upn
        }
        $baseUri = 'https://graph.microsoft.com/beta/deviceManagement/windowsAutopilotDeviceIdentities'
        $apDevices = foreach ($sn in $deviceSerial) {
            #make sure the device identity exists
            $deviceId = (Invoke-RestMethod -Method Get -Uri "$baseUri`?`$filter=contains(serialNumber,'$sn')" -Headers $script:authToken).value
            if ($deviceId) {
                Write-Host "Found device with id: $deviceSerial"
                $deviceId
                $body = @{
                    groupTag = $(if ($groupTag) { $groupTag } else { '' })
                }
                $update = Invoke-WebRequest -Method Post -Uri "$baseUri/$($deviceId.id)/updateDeviceProperties" -Body ($body | ConvertTo-Json -Compress) -Headers $script:authToken -UseBasicParsing
                if ($update.StatusCode -eq 200) {
                   Write-Host "Updated device: $deviceSerial with grouptag: $groupTag"
                }
                else {
                    throw "Web requested failed with status code: $update.statusCode"
                }
            }
        }
    }
    catch {
        $errorMsg = $_.Exception.Message
    }
    finally {
        if ($errorMsg) {
            Write-Warning $errorMsg
        }
        else {
            if ($sync) {
                Write-Host "Autopilot device sync requested.."
                Invoke-RestMethod -Method Post -Uri "https://graph.microsoft.com/beta/deviceManagement/windowsAutopilotSettings/sync" -Headers $script:authToken
            }
        }
    }
}
